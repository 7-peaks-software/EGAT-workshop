# Sample Queries for Testing

## Purpose

Use these sample queries to test your Document Inquiry Chatbot thoroughly. Queries are provided in both Thai and English.

---

## Category 1: Basic Document Search

### English Queries
```
1. "Show me all safety documents"
2. "I need IT policies"
3. "Find engineering standards"
4. "Where are the HR documents?"
5. "Show me project documentation"
6. "List all available policies"
7. "What documents do you have about training?"
8. "Find procedures for new employees"
```

### Thai Queries
```
1. "แสดงเอกสารความปลอดภัยทั้งหมด"
2. "ต้องการนโยบายด้าน IT"
3. "ค้นหามาตรฐานทางวิศวกรรม"
4. "เอกสารของฝ่าย HR อยู่ที่ไหน?"
5. "แสดงเอกสารโครงการ"
6. "แสดงรายการนโยบายทั้งหมด"
7. "มีเอกสารอะไรเกี่ยวกับการฝึกอบรม?"
8. "ค้นหาขั้นตอนสำหรับพนักงานใหม่"
```

**Expected**: List of relevant documents with links

## Testing Protocol

### For each query:
1. Enter the query
2. Wait for response
3. Check:
   - [ ] Response time < 5 seconds
   - [ ] Response is relevant
   - [ ] Sources are cited (if applicable)
   - [ ] Links work
   - [ ] Language is correct
   - [ ] Formatting is clean

## Quick Test Script

For rapid testing, use this sequence (5 minutes):

```
1. [Thai] "แสดงเอกสารความปลอดภัย"
2. [English] "I need IT policies"
3. [Specific] "Where is the employee handbook?"
4. [Information] "What are EGAT's working hours?"
5. [Fallback] "Show me documents about dragons"
```

All 5 should complete successfully to pass quick test.

## Advanced Test Scenarios

### Performance Test (100 queries)
Run these queries in rapid succession to test performance:
- Use a mix from all categories above
- Measure average response time
- Check for any errors or timeouts

### Stress Test (Concurrent users)
Have 10 users query simultaneously using queries from different categories

### Edge Case Test
- Very long query (500+ characters)
- Special characters: `@#$%^&*()[]{}|<>?`
- All caps: "SHOW ME DOCUMENTS"
- No punctuation: "show me safety documents please thanks"
