## Answer the following questions to see what you've learned.

**1. What is the purpose of creating a solution in Copilot Studio before building your agent?**

- [ ] To organize agents, connectors, and custom components in a structured environment.
- [ ] To change the branding and user interface of Copilot Studio.
- [ ] To assign user permissions and authentication roles.

<details>
<summary>Answer</summary>
To organize agents, connectors, and custom components in a structured environment.
</details>

---

**2. What happens when you disable general AI knowledge in an agent?**

- [ ] The agent will only respond using approved data sources that were added to the agent.
- [ ] The agent stops functioning until general knowledge is re-enabled.
- [ ] The agent will automatically switch to using the base Copilot agent instead.

<details>
<summary>Answer</summary>
The agent will only respond using approved data sources that were added to the agent.
</details>

---

**3. Where can you add data sources like Dataverse or public websites?**

- [ ] In the Knowledge tab of the agent's navigation menu in Copilot Studio.
- [ ] In the Topics section when editing conversation paths.
- [ ] In the Test pane's chat box.

<details>
<summary>Answer</summary>
In the Knowledge tab of the agent's navigation menu in Copilot Studio.
</details>

**4. What is a key reason to add both internal and external knowledge sources to your agent?**

- [ ] To ensure the agent can respond with both company-specific data and general contextual information.

- [ ] To make the agent, respond faster to questions by storing content in cache.

<details>
<summary>Answer</summary>
To ensure the agent can respond with both company-specific data and general contextual information.
</details>
